# Architecture Decision Records

Decision records capture durable choices that future sessions must not silently reverse.
Use a short `NNNN-title.md` file with Status, Context, Decision, Consequences, and
Supersedes sections. Supersede an accepted decision with a new record; do not rewrite
history to hide an earlier choice.
